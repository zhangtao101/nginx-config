# 版本说明

## 版本 1.0.5
**发布日期**: 2026-01-14

### 新增功能

#### 反向代理地址规范化
- **自动添加协议前缀**: 反向代理目标地址自动添加 `http://` 或 `https://` 前缀
- **协议验证**: 确保代理目标地址以正确的协议开头
- **智能补全**: 用户输入 `127.0.0.1:3000` 自动转换为 `http://127.0.0.1:3000`

#### 文件名自动补全
- **自动添加后缀**: 保存配置文件时，文件名如果没有 `.conf` 后缀会自动补全
- **用户友好**: 用户只需输入文件名即可，无需手动添加 `.conf`
- **避免重复**: 如果用户已输入 `.conf`，不会重复添加

### 改进优化
- **子配置文件列表样式修复**: 选中状态下删除按钮图标保持红色，颜色可见
- **代码规范修复**: 将 CommonJS 风格的 `require()` 导入改为 ES6 `import` 语法
- **配置生成优化**: 在生成 Nginx 配置时自动规范化代理目标 URL

### 技术改进
- 新增 `normalizeProxyUrl` 函数处理代理地址规范化
- 优化 `saveSubConfig` 函数实现文件名后缀自动补全
- 优化 `.file-item.active .delete-btn` 样式规则
- 修复 `src/main/store.ts` 中的 ESLint 错误

### 更新说明

#### 配置文件结构
```
nginx/
├── conf/
│   ├── nginx.conf          # 主配置文件(自动转换)
│   ├── nginx.backup.*.conf # 自动备份的原配置
│   └── conf.d/           # 子配置文件目录
│       └── *.conf        # 站点配置文件
├── logs/
│   ├── access.log         # 访问日志
│   └── error.log          # 错误日志
└── nginx.exe             # Nginx 可执行文件
```

#### 自动转换逻辑
1. 启动应用或选择 Nginx 目录时检测配置文件
2. 检查主配置是否包含 `include conf.d/*.conf`
3. 如果不是模块化配置:
   - 自动备份原配置文件
   - 创建新的模块化主配置文件
   - 创建 `conf.d` 目录
4. 用户可在 `conf.d` 目录中创建子配置文件

### 常见问题

**Q: 为什么我的配置文件被备份了?**
A: 应用在首次检测到非模块化配置时,会自动备份原配置文件并转换为模块化结构。备份文件位于 `conf/nginx.backup.{时间戳}.conf`。

**Q: 如何手动输入文件路径?**
A: 为了安全性考虑,根目录和证书文件路径必须通过系统弹窗选择,不支持手动输入。

**Q: 子配置文件在哪里?**
A: 子配置文件保存在 Nginx 安装目录的 `conf/conf.d/` 文件夹中,点击"子配置文件"按钮查看。

**Q: 如何回退到原配置?**
A: 可以在 `conf/` 目录中找到备份文件 `nginx.backup.{时间戳}.conf`,手动替换 `nginx.conf`。

**Q: 如何配置代理请求头?**
A: 在启用反向代理后,代理请求头区域会显示,可以手动添加或点击"添加常用请求头"按钮一键添加常用配置。

**Q: 为什么日志只显示部分内容?**
A: 为避免大文件导致页面卡顿,日志默认只读取最后 5000 行。如需查看更多内容,建议定期清空日志或使用日志轮转。

**Q: 为什么不能同时打开多个程序窗口?**
A: 为避免配置冲突,程序设置为单例模式,同时只能运行一个实例。尝试再次打开时会自动切换到已运行的窗口。
